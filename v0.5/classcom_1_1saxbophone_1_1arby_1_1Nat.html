Detailed Description

Arbitrary-precision unsigned integer type.

This is named after \(\mathbb{N}\), the set of Natural numbers, which this type models

Note
std::numeric_limits<Nat> is specialised such that most of the members of that type are implemented to describe the traits of this type.
Exceptions include any members of std::numeric_limits<> which describe a finite number of digits or a maximmum value, neither of which apply to this type as it is unbounded.
Exceptions
std::logic_errormay be thrown from most methods when the result of an operation leaves a Nat object with leading zero digits in its internal representation. Such cases are the result of bugs in this code and should be reported as such.

Member Typedef Documentation

◆ OverflowType

using com::saxbophone::arby::Nat::OverflowType = PRIVATE::StorageTraits::OverflowType

This is the smallest type guaranteed to be able to store the result of any product or sum of two values of the type used to store the number's digits.

Note
Consequently, this is the type used to represent Nat::BASE as that value is +1 beyond the upper bound for the type used to store the digits.

◆ StorageType

using com::saxbophone::arby::Nat::StorageType = PRIVATE::StorageTraits::StorageType

The type used to store the digits of this Nat object.

Note
The exact native type used for this is platform-specific:
  • Normally, it is the same as unsigned int
  • However, in the unlikely event that unsigned int is not smaller than uintmax_t, we pick the next smaller type (typically unsigned short)
